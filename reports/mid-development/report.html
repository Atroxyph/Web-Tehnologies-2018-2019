Project ASA Report

What our application will offer

Our application will mainly offer services concerning information related to security.

Namely, it will utilize the web services already provided by certain web applications such as Google Search, Google Safe Browsing, Shodan and HaveIBeenPwned - with the help of their APIs, which - upon proper usage, will provide to us the queried information in JSON format - which we will have to decode and utilize accordingly.

Furthermore, our app will provide a real-time notification system which will alert the user whenever new security vulnerabilities have arisen. The user will be notified inside the web application itself whenever he/she is logged in - otherwise the notification will be done by e-mail.

As a last foundational mechanism - we will also develop our own API for other developers to use in order to facilitate the usage of the information provided by *us*.

As a bonus feature - we will - instead of utilizing the Google Search API for our security information search service - we will instead engineer a Web Crawler that will be able to find relevant resources based on the user's search. Lastly, we will also store the user's past searches on our engine in order to provide a relevance-based search feature w/ the help of a Neural Network.

I will go on to explain briefly how each of these APIs will help us in our app.

  • Google Search

    We will utilize this application in order to be able to search articles and general information on the WWW with regard to security.

  • Shodan

    Shodan - also known as "Dark Google" will provide us the ability to search for security vulnerabilities/exploits and inform us certain properties of theirs such as their author, the platform the exploit is intended for as well as the code itself (be careful if you plan on using it!).

  • Google Safe Browsing

    This will consist of a search form where a user will be able to introduce several website URLs to check if their are known to be malicious or not.

  • HaveIBeenPwned

    This one is pretty self-explanatory. We will provide a simple search form to the user where he/she will be able to introduce his/her e-mail to check whether it is known to have been compromised in a past security breach.
